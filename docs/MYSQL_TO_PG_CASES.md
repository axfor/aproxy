## 数据类型转换

### 整数类型

| MySQL 类型 | PostgreSQL 类型 | 测试状态 | 说明 |
|-----------|----------------|---------|------|
| `TINYINT` | `SMALLINT` | ✅ | 8位 → 16位 |
| `TINYINT(1)` | `BOOLEAN` | ✅ | 布尔值检测 |
| `SMALLINT` | `SMALLINT` | ✅ | 16位整数 |
| `MEDIUMINT` | `INTEGER` | ✅ | 24位 → 32位 |
| `INT` / `INTEGER` | `INTEGER` | ✅ | 32位整数 |
| `BIGINT` | `BIGINT` | ✅ | 64位整数 |
| `TINYINT UNSIGNED` | `SMALLINT CHECK(value >= 0)` | ✅ | 无符号转换 |
| `INT UNSIGNED` | `BIGINT CHECK(value >= 0)` | ✅ | 无符号转换 |
| `BIGINT UNSIGNED` | `NUMERIC CHECK(value >= 0)` | ✅ | 超大整数 |

**测试用例:**
```sql
-- MySQL
CREATE TABLE test_int (
    id INT AUTO_INCREMENT PRIMARY KEY,
    tiny TINYINT,
    small SMALLINT,
    medium MEDIUMINT,
    big BIGINT,
    is_active TINYINT(1),
    unsigned_val INT UNSIGNED
);

-- PostgreSQL (转换后)
CREATE TABLE test_int (
    id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    tiny SMALLINT,
    small SMALLINT,
    medium INTEGER,
    big BIGINT,
    is_active BOOLEAN,
    unsigned_val BIGINT CHECK(unsigned_val >= 0)
);
```

## SQL 语法转换

### 1. 标识符引号

| MySQL | PostgreSQL | 测试状态 |
|-------|-----------|---------|
| ``` `identifier` ``` | `"identifier"` | ✅ |
| ``` `table`.`column` ``` | `"table"."column"` | ✅ |

**测试用例:**
```sql
-- MySQL
SELECT `id`, `name` FROM `users` WHERE `status` = 'active';

-- PostgreSQL (转换后)
SELECT "id", "name" FROM "users" WHERE "status" = 'active';
```

### 2. 占位符

| MySQL | PostgreSQL | 测试状态 |
|-------|-----------|---------|
| `?` | `$1, $2, $3, ...` | ✅ |

**测试用例:**
```sql
-- MySQL
SELECT * FROM users WHERE id = ? AND status = ?;

-- PostgreSQL (转换后)
SELECT * FROM users WHERE id = $1 AND status = $2;
```

### 3. LIMIT 语法

| MySQL | PostgreSQL | 测试状态 |
|-------|-----------|---------|
| `LIMIT offset, count` | `LIMIT count OFFSET offset` | ✅ |
| `LIMIT count` | `LIMIT count` | ✅ |

**测试用例:**
```sql
-- MySQL
SELECT * FROM users LIMIT 10, 20;
SELECT * FROM users LIMIT 10;

-- PostgreSQL (转换后)
SELECT * FROM users LIMIT 20 OFFSET 10;
SELECT * FROM users LIMIT 10;
```

### 4. INSERT ... ON DUPLICATE KEY UPDATE

| MySQL | PostgreSQL | 测试状态 |
|-------|-----------|---------|
| `INSERT ... ON DUPLICATE KEY UPDATE` | `INSERT ... ON CONFLICT ... DO UPDATE` | ✅ |

**测试用例:**
```sql
-- MySQL
INSERT INTO users (id, name, email) VALUES (1, 'Alice', 'alice@example.com')
ON DUPLICATE KEY UPDATE name = VALUES(name), email = VALUES(email);

-- PostgreSQL (转换后)
INSERT INTO users (id, name, email) VALUES (1, 'Alice', 'alice@example.com')
ON CONFLICT (id) DO UPDATE SET name = EXCLUDED.name, email = EXCLUDED.email;
```

### 5. REPLACE INTO

| MySQL | PostgreSQL | 测试状态 |
|-------|-----------|---------|
| `REPLACE INTO` | `INSERT ... ON CONFLICT ... DO UPDATE` | ✅ |

**测试用例:**
```sql
-- MySQL
REPLACE INTO users (id, name) VALUES (1, 'Bob');

-- PostgreSQL (转换后)
INSERT INTO users (id, name) VALUES (1, 'Bob')
ON CONFLICT (id) DO UPDATE SET name = EXCLUDED.name;
```

| `COLLATE=utf8mb4_general_ci` | (移除) | ✅ |

**测试用例:**
```sql
-- MySQL

    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    UNIQUE KEY uk_email (email)

    id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_name ON users(name);
```

## 已知限制

### 1. 无法支持的功能

- **存储过程和函数**: MySQL 和 PostgreSQL 语法差异太大,需要手动重写
- **触发器**: 语法不同,需要重写
- **全文索引**: `FULLTEXT INDEX` 在 PostgreSQL 中使用不同的机制
- **空间数据类型**: `GEOMETRY`, `POINT` 等需要 PostGIS 扩展
- **分区表**: 语法差异,需要重写
- **外键级联**: 某些级联选项不完全兼容

### 2. 行为差异

- **自增列重置**: MySQL 的 `AUTO_INCREMENT` 重置行为与 PostgreSQL 的 `SERIAL` 不同
- **字符串比较**: MySQL 默认不区分大小写,PostgreSQL 区分大小写
- **NULL 值排序**: MySQL 和 PostgreSQL 的 NULL 值排序规则不同
- **除零行为**: MySQL 可能返回 NULL,PostgreSQL 会抛出错误
- **日期时间精度**: 微秒精度处理可能不同

### 3. 需要注意的地方

- **引号使用**: PostgreSQL 严格区分单引号(字符串)和双引号(标识符)
- **类型转换**: PostgreSQL 对类型转换更严格
- **字符集**: PostgreSQL 使用不同的字符集管理方式
- **排序规则**: COLLATE 支持不同
